# Objetos de negocio iniciales

Los siguientes nombres identifican conceptos funcionales del frontend. No implican una tecnología concreta de implementación, una entidad CDS ni un endpoint OData.

| Business object | Propósito inicial |
| --- | --- |
| `Labels` | Gestión de etiquetas hospitalarias y COMEX. |
| `QuarantinePackages` | Consulta y tratamiento futuro de bultos en cuarentena. |
| `PackageApproval` | Consulta futura de bultos aprobados o rechazados. |
| `Traceability` | Seguimiento de series, movimientos y verificaciones. |
| `LabelReprint` | Gestión futura de reimpresiones de etiquetas. |
| `ScannerVerification` | Verificación futura mediante lectura de scanner. |
| `Reports` | Consultas e informes funcionales. |
| `Settings` | Parámetros y configuración de la aplicación. |
| `ApplicationLogs` | Consulta de eventos y registros de la aplicación. |

La definición detallada de contratos, atributos, reglas, autorizaciones y operaciones se realizará cuando se diseñe el backend RAP/OData.
